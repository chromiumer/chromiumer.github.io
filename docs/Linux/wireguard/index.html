wireguard

说明：wireguard使用udp协议通信，服务端和客户端只通过交换公钥实现数据加解密。

服务端安装​

Ubuntu 24.04​

sudo apt update
sudo apt install wireguard -y

CentOS7/8​

##centos7
yum -y install yum-utils epel-release
yum-config-manager --setopt=centosplus.includepkgs=kernel-plus --enablerepo=centosplus --save
sed -e 's/^DEFAULTKERNEL=kernel$/DEFAULTKERNEL=kernel-plus/' -i /etc/sysconfig/kernel
yum -y install kernel-plus wireguard-tools
reboot
##centos8
yum -y install elrepo-release epel-release
yum -y install kmod-wireguard wireguard-tools

加载内核模块​

modprobe wireguard && lsmod | grep wireguard
wireguard             217088  0
ip6_udp_tunnel         16384  1 wireguard
udp_tunnel             16384  1 wireguard
##(如无法加载内核模块尝试重启或使用 wireguard-go 
##https://github.com/WireGuard/wireguard-go)

git clone https://git.zx2c4.com/wireguard-go
cd wireguard-go
make

## 创建虚拟wg0网卡
./wireguard-go wg0

生成密钥对​

cd /etc/wireguard/
wg genkey | tee private.key | wg pubkey > public.key
私钥文件: private.key
公钥文件: public.key

配置文件​

服务端​

/etc/wireguard/wg0.conf
cat <<EOF>> /etc/wireguard/wg0.conf
[Interface]
Address = 1.1.1.1/24
SaveConfig = true
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE;
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE;
ListenPort = 666
PrivateKey =  private.key //私钥文件内容
EOF

## 在服务端添加客户端授信
wg set wg0 peer client-public-key allowed-ips 1.1.1.2/32

##删除授信
##wg set wg0 peer client-public-key remove

客户端​

/etc/wireguard/wg0.conf
cat <<EOF>> /etc/wireguard/wg0.conf
[Interface]
PrivateKey = prikey_of_local
Address = 1.1.1.2/24
DNS = 223.5.5.5, 223.6.6.6

[Peer]
PublicKey = pubkey_of_server
AllowedIPs = 1.1.1.0/24
Endpoint = server_ip:1521
PersistentKeepalive = 30
EOF

开启路由转发​

/etc/sysctl.conf
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
sysctl -p

iptables -t nat -A POSTROUTING -j MASQUERADE

服务操作​

启动
wg-quick up wg0

关停
wg-quick down wg0

检查状态
wg
interface: wg0
  public key: server-public-key
  private key: (hidden)
  listening port: 666
xxx xxx xxx xxx
xxx xxx xxx xxx
